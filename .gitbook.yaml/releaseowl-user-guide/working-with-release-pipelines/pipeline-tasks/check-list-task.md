# Check List Task

A **Checklist Task** is used to perform **pre-release validation and compliance checks** before proceeding with deployment. It allows teams to define and verify mandatory steps, approvals, and conditions that must be completed prior to deployment. A **Check List Task** can be added as follows:

<figure><img src="../../../.gitbook/assets/image (1760).png" alt=""><figcaption></figcaption></figure>

Fill in the required details:

| **Field**                      | **Description**                               | **Action to Perform**                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                    |
| ------------------------------ | --------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **Name**                       | Name of the checklist task                    | Enter a valid name using letters, numbers, underscores (\_) or periods (.) (e.g., `Check_List_SAS`)                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Description**                | Purpose of the checklist                      | Any message that is to be conveyed to the task performer can be mentioned here.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Assign To**                  | User responsible for completing the checklist | <p>Select a user from the available list:                                      <br><strong>User:</strong><br>Select a specific user from the available list. The selected user will be responsible for completing the checklist task. <br><strong>Role:</strong><br>Any user with the selected role will be able to complete the checklist task. This is useful when checklist execution is not tied to a specific individual but to a functional role (e.g., QA, Release Manager).<br><strong>Custom:</strong><br>The task can be completed by any user associated with the <strong>custom-defined roles assigned to each component</strong>. This provides flexibility to assign different roles for different components involved in the checklist validation process.      <br><strong>Promoter:</strong><br>The assigned user/role/custom-mapped user acts as the promoter who completes all checklist items.  </p> |
| **Disable Email Notification** | Control email alerts                          | Users will not receive the approval email link but will get a notification only about the final status (approved/rejected).                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| **Approval Message Required**  | Enforce comments during completion            | When **Approval Comment Required** is enabled at the task level, approvers must enter a comment while performing **Approve** or **Reject** actions from the **My Tasks** section. The action cannot be completed without a comment.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Checklist Items**            | Define validation steps                       | Click **+ Add** to create checklist items                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Checklist Item Name**        | Name of each validation step                  | Enter clear names for each checklist item.                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
| **Mandatory**                  | Indicates compulsory checks                   | Enable to ensure the step must be completed before proceeding                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| **Evidence Required**          | Proof for compliance                          | Enable to require supporting documents/screenshots as evidence                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                           |

#### **Completing the Checklist Task**

1. After adding the Checklist Task to the User Story, **save and promote** the User Story.
2. Navigate to the **My Tasks** section.
3. Locate the assigned task and click on **Approve / Reject** to open it.

<figure><img src="../../../.gitbook/assets/image (2) (1) (1) (1).png" alt=""><figcaption></figcaption></figure>

* Inside the task, you will see the **Checklist items** that must be completed before promoting to the next stage.
* For each checklist item:
  * If marked as **Mandatory (★)**, it must be completed.
  * Click on the item and **upload the required file** using the **Browse** option.
  * After uploading, **mark the checklist item as Done**.
  * Add comments if required.
* Ensure all mandatory checklist items are completed.
* Click **Approve** to proceed to the next stage (or **Reject** if the task is not satisfied).

<figure><img src="../../../.gitbook/assets/image (3) (1) (1) (1).png" alt=""><figcaption></figcaption></figure>

* After performing **Approve** or **Reject**, the executed checklist details can be viewed in the **Pipeline Activity** section.
